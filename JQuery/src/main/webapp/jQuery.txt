JSON(JavaScript Object Notation) 타입
=> 자바스크립트 객체 표현식

[장점]
- 개방형 표준 데이터 포멧 (거의 모든곳에서 해당 타입의 데이터 사용가능)
- [json] key : value 쌍으로 정보를 저장 => 데이터 저장크기 적음(전송 속도가 빠르다는 의미를 가짐)
  [xml]  <tag>key</tag> , <tag>value</tag>
- ajax 응답결과가 다른 도메인주소라도 요청을 처리가능
- 주로 웹애플리케이션에서 사용
- 대부분의 플랫폼에서 JSON타입을 지원 / 브라우저에서 json 파서 포함

[단점]
- 자바스크립트 문법을 따름. 즉 예민함(엄격함)
- 자바스크립트 코드를 포함가능(불법적인코드도 숨길 수 있음)

[문법]
- JSON 객체 중괄호 { }로 표기
- JSON 배열 대괄호 [ ]로 표기

- 속성(key) : 값(value)로 저장
- 속성은 쌍따옴표(")f로 묶어서 표시
- 값은 데이터 타입에 따라서 표기법이 변경
- 속성의 개수가 여러개 일때 ,(콤마)로 구분 (마지막 요소는 , 안씀)

- 정수: 2진수, 8진수, 16진수는 사용 불가
- 실수: 1) 고정 소수점: 3.14, -9.653
        2) 부동 소수점: 1e4, 6.7E-5
- boolean: true, false;
- null
- 문자열: "" 묶여있는 0개 이상의 유니코드 문자 
- 이스케이프 문법 지원: (\n, \t)
- 배열

- 날짜정보 처리시 주의 
  => 1) 정수타입의 날짜정보(ms)
     2) 문자 타입의 날짜 정보 "2025-09-23"(꼭 쌍따옴표 찍어서 표현해야 함)
